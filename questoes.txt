1. O que é um repositório no Git e qual a sua importância?
    O repositório Git serve como um backup, uma ferramenta que nos ajuda no versionamento dos códigos. Ele é importante para sempre termos uma versão "pronta" e funcionando do projeto.
2. Qual a função do Git Bash?
    Criarmos um repositório.
3. O que faz o comando git add e qual a diferença entre git add <arquivo> e git add .?
    O "git add" serve para adicionarmos coisas/arquivos ao nosso repositório. Com o "git add <arquivo>", adicionamos o arquivos específico a qual colocamos o nome. Com o "git add ." adicionamos todos os arquivos quais foram modificados.
4. Por que utilizamos o ponto (.) no comando git add .?
    O "." se refere a pasta que estamos, logo, quando escrevemos "git add .", pegamos todos os aquivos que estão modificados na pasta em que estamos.
5. Explique o comando git commit e por que é importante criar uma mensagem descritiva.
    O "git commit" serve para criamos uma "marca" no tempo, para salvarmos aquele código. A descrição do commit é fundamental para sabermos quais foram as últimas alterações feitas naquele código que está salvo.
6. Qual a função do comando git push?
    O "git push" serve para mandar o commit para algum lugar (para o github, por exemplo)
7. Qual a importância de criar commits frequentes durante o desenvolvimento de um projeto?
    Para nunca "perdermos" o que já foi feito e se caso fizermos um teste e der errado, temos uma cópia salva no commit das últimas alterações que fizemos. Com commits frequentes "perdemos" menos linhas de código.
8. Explique a função das tags HTML e como elas estruturam uma página.
    As tags HTML servem como uma marcação, para indicar o que é o conteúdo que está dentro.
9. O que são atributos em uma tag HTML e qual a sua função?
    Atributos em uma tag HTML são coisas que indicam como aquela tag irá se comportar. Uma exemplo é o atributo "class=''", que serve como se fosse uma "variável" para estilizarmos no CSS.
10. O que é o Emmet e como ele facilita o desenvolvimento de páginas HTML?
    O Emmet é um facilita a escrita dos códigos, otimizando tempo de escrita e evitando erros de digitação.
11. O que é o box-sizing no CSS e por que usamos ele no reset inicial?
    O box-sizing é uma propriedade do CSS onde setamos como será calculado o tamanho do elemento.
12. Por que é importante utilizar tags semânticas no HTML?
    É importante utilizar as tags semânticas para se entender o que é cada conteúdo do HTML. Também é importante para uma questão de acessibilidade.
13. O que são media queries no CSS e qual a sua função?
    A função do media queries é aplicar um estilo específico para tal ocasião aplicado ao media.
14. Qual a função do GitHub Pages?
    O Github Pages funciona como um servidor, onde podemos "hospedar nossos web sites" e assim simular como ele seria.
15. O que é o Flexbox no CSS e para que ele é utilizado?
    É uma propriedade do CSS que facilica o alinhamento de itens.
16. Qual a diferença entre justify-content e align-items em um container Flexbox?
    O "justify-content" alinha os itens no eixo principal, e o "align-items" no eixo secundário.
17. Como o flex-grow funciona e qual o seu propósito no layout Flexbox?
    O flex-grow é usado para controlar como os itens dentro de um Flexbox crescem, ocupando o espaço extra disponível dentro de um contêiner. Permite que os itens cresçam para preencher o espaço que resta.
18. Explique a diferença entre flex-direction: column; e flex-direction: row;.
    O flex-direction: column; alinha os itens da Flexbox em coluna, enquanto o flex-direction: row; alinha os itens em linha.
